Descripción del Proyecto

Este proyecto corresponde al desarrollo y simulación de un Autómata con Pila (PDA) en entorno web. El objetivo es diseñar un autómata capaz de reconocer un lenguaje no regular, implementando su funcionamiento mediante una página web interactiva.

Lenguaje: L = { aⁿ bᵐ x y* cᵐ aⁿ | n ≥ 2, m ≥ 1 }

Objetivo General

Desarrollar una simulación web funcional que represente el comportamiento de un Autómata con Pila (PDA) diseñado para reconocer el lenguaje L, permitiendo el ingreso de cadenas, la visualización de la pila y la verificación de aceptación o rechazo.

Definición Formal del PDA



A = (Q, Σ, Γ, δ, q₀, Z₀, F)

Q  = { q₀, q₁, q₂, q₃, q₄ }
Σ  = { a, b, c, x, y }
Γ  = { A, B, Z }
q₀ = q₀
Z₀ = Z
F  = { q₄ }



    
Función de Transiciones δ



δ(q₀, a, Z) = (q₀, AZ)
δ(q₀, a, A) = (q₀, AA)

δ(q₁, b, A) = (q₁, AB)
δ(q₁, b, B) = (q₁, BB)

δ(q₂, x, B) = (q₂, B)
δ(q₂, y, B) = (q₂, B)

δ(q₃, c, B) = (q₃, λ)

δ(q₄, a, A) = (q₄, λ)

Descripción Teórica del PDA

    

      
        
ElementoDescripción

      
      
        
Σ (Alfabeto){ a, b, c, x, y }

        
Γ (Símbolos de pila){ A, B, Z }

        
Q (Estados){ q₀, q₁, q₂, q₃, q₄ }

        
q₀Estado inicial

        
Z₀Símbolo inicial de pila

        
F{ q₄ }

      
    


    

      El autómata empuja A por cada a, luego B por cada b. Al llegar a x 
      continúa leyendo posibles y. Posteriormente, por cada c elimina un B,
      y finalmente elimina las A cuando aparecen las últimas a.
      La cadena se acepta si la pila vuelve a quedar únicamente con Z.
